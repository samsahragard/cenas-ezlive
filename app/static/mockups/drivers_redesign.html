Desktop view — design decisions:
  • No hamburger button rendered — sidebar is always-visible at ≥1024px, so hamburger is redundant. Hidden via @media (max-width: 1024px) { .menu-toggle { display: flex } } else display: none in base_dashboard chrome. Closes the "hamburger covering page title" issue at root rather than padding-hacking around it.
  • Page title clears cleanly — sits flush against the main-pane padding (26px left). Title font-size: 22px / weight 600 / color var(--ck-text).
  • Active / Inactive tabs use the V2 .ezm-tab underline pattern from ez_manage.html. Active state: 2px var(--ck-gold-bright) bottom border + rgba(0,0,0,0.55) recess bg + var(--ck-text-hover) text. Counts in dim parens beside each label so the operator knows magnitudes before clicking.
  • STATUS column removed — redundant with tab filter.
  • Topbar meta line — "Tomball · 12 active · 4 inactive" sits at-a-glance right of title, replacing the lost STATUS column information density.

Phone view — design decisions:
  • Hamburger = 48×48 touch target. iOS HIG + Material a11y standard (current ~32 was too small). Background var(--ck-gold-soft), border 1px solid var(--ck-gold), radius 10px. Hover bg deepens to rgba(217,164,54,0.28). Bars are var(--ck-gold-bright) 22×2.5px, gap 5px — high-contrast against the dark glass.
  • Hamburger lives INSIDE the topbar as the leftmost flex element (12px gap to title). NOT position: fixed. This is the structural fix — collision with page title becomes architecturally impossible regardless of viewport width or page content. Hamburger is also predictably present at the top of every mobile page (visible affordance, not floating-overlay surprise).
  • Tabs row uses overflow-x: auto for narrow widths. With two tabs they fit; future 3rd/4th tab states scroll horizontally.
  • Driver-meta hidden on phone (rating / delivery count) to keep rows scannable at small width. Tap on a row navigates to the driver-detail page where full meta lives.
Tokens, rationale, implementation notes
  • All colors from V2 --ck-* tokens defined in app/static/css/sidebar.css:8-50. No raw hex outside :root. This closes design-debt items §5.7 (V1↔V2 token bridge) and §5.4 (breakpoint alignment) from the design-system-reference doc when shipped.
  • Hamburger fix is cross-cutting — base_dashboard.html chrome change, every authenticated page benefits, not Drivers-specific.
  • Tabs pattern is .ezm-tab reuse — no new tabs class. Don't introduce a fourth tab pattern when V2 already has one that fits.
  • Filter shape: recommend server-side /drivers?status=active|inactive querystring (shareable URLs, cleaner state, doesn't slow growing rosters) over client-side row hiding.
  • Breakpoint: standardize on --ck-bp-mobile: 1024px. Currently dashboard.css has 900px + 720px; sidebar.css has 1024px. Pick 1024 per the V2 token system.
  • Three-gate (samai): visual confirm Drivers + one other page (suggest home dashboard) on phone AND desktop + hamburger 48×48 touch target measured.
  • Animation: hover transitions use 200ms var(--ck-ease) (the cubic-bezier(0.22, 0.8, 0.24, 1) brand motion).
