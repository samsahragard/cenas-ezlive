Desktop view — design decisions:
  • No hamburger at ≥1024px — sidebar is always-visible, hamburger redundant. Hidden via @media (max-width: 1024px) { .menu-toggle { display: flex } } in base_dashboard chrome.
  • Topbar = title + location pill only (v2 change). Tab counts carry the active/inactive magnitudes; no duplication.
  • Active / Inactive tabs (v3) — pill pattern matching V2 .lg-nav (Legal subsystem nav). Inactive = pure white text on transparent bg + gold-soft border. Active = dark text on var(--ck-gold-bright) pill bg with 14px glow. Per Sam #2485 via cena.
  • Driver rows are tappable (v2 change). First row shows the hover state — gold-soft bg + chevron-right nudged + colored gold. Real implementation: :hover drives this; row click → /drivers/<id>.
  • Status-meta inline on desktop (rating · deliveries · last-shift) — desktop has the horizontal room, so all three pieces stay on the row.

Phone view — design decisions:
  • Hamburger = 48×48 touch target inside the topbar (not position: fixed). Bg var(--ck-gold-soft), border 1px solid var(--ck-gold), radius 10px. Bars are var(--ck-gold-bright) 22×2.5px.
  • One status-meta line per row (v2 change). "on shift" in var(--ck-status-open) for live drivers; "last shift Xh ago" in soft cream for the rest. Rating + delivery count moved to the driver-detail page — status > stats for live ops (GMs during lunch rush).
  • Tappability affordance (v2 change) — row hover state + ti ti-chevron-right. First row shown in hover state. On tap → /drivers/<id>.
  • Tabs row uses overflow-x: auto for narrow widths.
Tokens, rationale, implementation notes
