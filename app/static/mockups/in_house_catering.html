In-House Catering Quote Builder — v1 mockup

dck · 2026-05-18 · per Sam #837 item 16 + cena #1031 + aick relay #2875

What this mockup demos

  • Full page chrome per cena #1050: left app-sidebar with new "In-House Catering" entry under Today, topbar with breadcrumbs + location toggle, 3-col content (category nav | item grid | cart drawer).
  • STAFF tool optimization per cena #1050: items show $0.00 by default, ezCater reference price shown as ghost hint ("ref $21.49"), modifier modal opens with prominent custom-price input at top + ezCater ghost ("ezCater charges $X for the same package"). No customer-self-serve patterns.
  • Item cards: photo slot (placeholder per cena #1050 v1 — no photos yet, room for future image-gen), label, description, $0 quote price + per-person/per-item unit, ezCater ref-price ghost right-aligned, "Most ordered" gold badge top-left when applicable, min-qty serves note.
  • Modifier modal: custom-price input prominent (gold-bordered top section), required modifier groups (radios for choose-one, checkboxes for choose-N), upcharge inline ("+$0.49"), qty stepper + per-item special instructions, Add-to-quote button shows live total. Tray/Individual Packaging modifiers OMITTED per Sam #910.
  • Cart drawer (340px right rail): line items with summary modifiers, click to edit, trash to remove. Subtotal + estimated tax + total. Customer form below (name/email/phone/event_date/setup_required/notes). 3 CTAs with locked hierarchy: Quote (blue primary) > Pay Now (green secondary) > Pay Later (disabled, PCI-pending tooltip) per cena #1050.
  • Mobile responsive: at ≤1024px, sidebar + category nav + cart drawer collapse; main column becomes the item grid; bottom-bar shows "Quote total · N items" with View cart CTA opening drawer.

Alignment with ck #1049 production plan

This mockup is the visual reference for ck's production build per cena #1052. Specific alignments:

  • Route: /<store>/in-house-catering (per aick backend #2875 — page + menu.json + quote + pay-now endpoints already live at 8311c04)
  • Sidebar: new entry under Today (matches Notifications #2825 placement), label "In-House Catering", icon ti-shopping-cart
  • POST body shape per ck #1049 cont 6/9: {customer, lines, store, location, notes}
  • Email-on-Quote is currently STUB per aick — Quote button creates draft row, SMTP wiring is wave-2
  • Pay Later button stays disabled with tooltip until Sam's PCI decision (Sam #1038)

Decisions deferred to ck implementation

  • Category default order on nav — mockup shows the source-of-truth order from app/data/in_house_catering_menu.py; ck should match unless usage data later suggests reordering by frequency
  • Modal close behavior on Add — mockup auto-closes; ck can wire a "Add another" variant if staff workflow benefits
  • Cart edit flow — mockup shows click-to-reopen-modal-with-prior-selections-prefilled; ck implements modal state preservation
  • Tax estimate — mockup shows 8.25% TX state tax as illustration; backend authoritative

v2 follow-up backlog (post-Sam-approval)

  • Item photos per cena #1050 — image-gen pipeline same shape as produce v1.2 (Sam-provided refs → dck slice + match → app/static/img/in-house-catering/<slug>.jpg). Single-line markup swap when ready.
  • Quick-add keyboard shortcuts — STAFF tool optimization (cena #1050): item card hover → number-key to add 1, shift+number to add 5, modifiers pre-set to "Most popular" defaults. Phase 2 polish.
  • Saved customer search — populate customer form by selecting from prior quote customers (autocomplete on name/email). Reduces typing for repeat clients.
  • Quote draft list page — separate route showing all open drafts so staff can resume builds; today every refresh loses the cart. (Backend has the Quote model from aick 8311c04 already.)
  • Multi-store unified view — if Copperfield adds In-House Catering with its own menu, top-bar location toggle filters/switches.

Approval workflow

Standard pattern: mockup ships to /partner/developer/samples as a card, Sam reviews + approves via the workflow (samples_watch.py at b47cd35 will fire on approval). On approve, ck implements production frontend against this canonical visual reference; aick wires the email-send wave-2 and Sam's PCI decision unblocks Pay Later wave-3.
