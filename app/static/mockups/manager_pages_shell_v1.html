Manager Pages — shared shell pattern, v1 (text-heavy)

14 pages share this one shell · iterate fields after Sam sees live

What the shared shell defines (apply once, all 14 pages benefit)

  • Page header: icon + title + entry count + access pill (KM+ green vs GM+ red) + "New entry" CTA right-aligned
  • Filter row: search input + time-range chip seg (Last 30d / 7d / Today / All) + author-scope chip seg (All / Mine only) + count meta on right
  • 2-col body: list (left) + detail/form (right). Click a list item → detail view loads on right. "New entry" CTA flips right pane to form mode.
  • List item: author + date in muted header row + bold title + 2-line snippet (CSS -webkit-line-clamp). Selected state = left gold border + soft bg.
  • Type tag (optional, surface-specific): small color-coded pill on list items where a per-entry category exists (Incident Reports uses Injury/Theft/Complaint tags; Daily Manager Log doesn't because no per-entry type).
  • Detail view: mode label + title + author/date/store meta + read-only body in monospace-ish read-value block + actions row (Edit / Back / Delete-danger).
  • Form view: Title input + Type select (where applicable) + Body textarea + field-help tooltips + Save/Cancel actions.
  • Mobile collapse: list stacks above detail, list max-height 280px so detail stays visible. Filters wrap to column.

14 pages this shell serves (per aick #29xx + correction #3018)

All 14 share the SAME audience gate per Sam clarification at /sam/chat #1109 (no per-page carveouts, no GM+-only exceptions):

  • 1. Daily Manager Log · 2. Shift Handoff · 3. Incident Reports · 4. Supply Requests · 5. Daily Goals
  • 6. Staff Feedback · 7. Pre-shift Checklist · 8. Close-of-day Audit · 9. Recipe Page · 10. Attendance Tracking
  • 11. Interview Surface · 12. Training Records · 13. Maintenance Requests · 14. Employee Counseling

Approach A v1 (text-heavy, per Sam direction)

  • Every page = same form-and-list shape. Title + free-text body + auto-stamped author + date. Type-tag where useful (Incident Reports). No structured fields, no checklists, no attachments in v1.
  • Fields iterate after Sam sees the live page. If Pre-shift Checklist needs actual checkbox items, that's v2 spec. v1 ships the text-heavy version and lets pattern emerge from real use.
  • "Mine only" filter on author scope makes it cheap for an individual manager to find their own entries vs scanning the team's. Defaults to "All authors" so handoff visibility is preserved.

Access gating (one gate for all 14 pages, per Sam #1109 + cena #1111)

  • Writers (per their store): GM, Kitchen Manager, Assistant KM, FOH Manager. cena confirmed foh_manager role exists in permissions.py at rank 2.
  • Readers (across all stores): Partner, Corporate.
  • Locked out: Expo. No access to Manager pages at all.
  • Code shape (cena suggested): single helper is_store_manager(user) returns True for {gm, km, asst_km, foh_manager}. Applied uniformly to all 14 page route handlers. Partner/corporate read-all uses the existing read-tier helper.
  • Access pill in page header reads "GM / KM / Asst KM / FOH manager" — same on every page since the gate is uniform. Single visual reminder of who's writing; no per-page differentiation.
  • Sidebar entries: aick wires the 14 sidebar entries under Manager section after Sam approves this mockup + ck builds production + aick wires backend. Existing 7 entries shipped at 12733ad get the other 7 added.

Decisions deferred to ck implementation

  • List sort order — mockup shows newest first; ck implements DESC by created_at. Saved-state pagination follows existing pattern.
  • Search backend: simple ILIKE on title+body works for v1 (per-store small corpus). Postgres full-text if corpus grows.
  • Cross-store rollup view for partner: show all stores' entries in one list, with per-row store badge. Mockup doesn't demo since it's role-gated; ck wires when GM/partner-tier session probes the page.
  • Optimistic save: Save button sends POST; form disables until 200 returns. No autosave (free-text edits aren't critical-loss).

v2 backlog (post-Sam-approval, iterate after real use)

  • Per-page structured fields where patterns emerge (Pre-shift Checklist → actual checkbox items, Attendance Tracking → per-employee daily grid, Recipe Page → photo + ingredient list)
  • Attachments — photos for Incident Reports + Maintenance Requests + Recipe Page
  • Mentions/assignment — @-mention an employee in Staff Feedback or Daily Goals; assigned items show in their personal queue
  • Recurring entries — Pre-shift Checklist + Close-of-day Audit could auto-create blank entries each morning/evening so they're prompts not write-from-scratch
  • Filter by date range custom picker — beyond Last 30d / 7d / Today chip presets
  • Export to PDF — Incident Reports + Employee Counseling for compliance/legal retention
