Source pattern

Uber Eats Manager. Every metric in their analytics view carries a delta vs. the prior 7d / 12w / 12mo window, with the comparison range user-selectable at the top of the screen rather than per-card. The chip turns a static number into a verdict ("good day" vs. "slow day") without a second click. The single global range selector beats per-card range selectors because most operators want to compare ALL their numbers to the same baseline — not slice one metric one way and another differently.

Why it works

  • Cognitive load: zero. Operator doesn't have to remember last Tue's covers — the chip does the math + the comparison + the verdict in one glance.
  • Pre-attentive color. Green up / red down / gray flat. Operator can scan the dashboard with peripheral vision and spot anomalies (red food-cost %) without reading anything.
  • Single global control. No drift between cards. One range pick = one consistent picture.

How it adapts to Cenas

  • Per-store dashboards: hang the chip under every KPI on /Tomball and /Copperfield home pages. Initial set above: Covers / Ticket Avg / EzCater Orders / Driver Runs / Avg Delivery Time / Food Cost %. Add Labor % / Top SKU as they become available.
  • Range selector at top. "Yesterday / Last [same day-of-week] / Last 7 days / Last 30 days / Last year same week" — the same-day-of-week comparison is the one Tex-Mex managers care about most (Tue ≠ Sat).
  • Color semantics flip per metric. Higher Covers = good (green up). Higher Food Cost % = bad (red up). The mockup demos both — see Food Cost card showing red "up" because the direction is bad even though the value rose.
  • Sparkline is bonus, not the headline. Optional small inline chart per card — shows trend trajectory at a glance. ~30px high, no axes, no labels. Cheap retention of context.

Implementation cost

Backend: extend each KPI query to also fetch the comparison-period value (parameterized by the range selector). Frontend: one new .kpi-card component (~30 lines CSS, ~15 lines template), one range-bar component (~20 lines), one delta-classifier helper (returns up/down/flat + direction-is-good boolean per metric). Total: ~2 days for one engineer to ship across both store dashboards.

Citation

Pattern observed in Uber Eats Manager analytics screens. See: uber.com/us/en/blog/restaurant-manager/ + Uber Eats Manager performance-data docs. Pattern also exists in Square Dashboard + Toast Analytics with smaller surface area.
