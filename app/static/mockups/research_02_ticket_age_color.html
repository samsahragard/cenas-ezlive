Source pattern

Toast KDS. Kitchen Display ticket headers shift color as the ticket ages, and new/changed tickets briefly flash. Kitchen reads urgency peripherally — no time math, no reading timestamps. This is the single most universally-praised pattern in restaurant ops UX. Toast lets operators customize the thresholds per their service standards, but the color escalation itself is non-negotiable in the product.

Why it works

  • Pre-attentive color processing. Humans see "that one's red" before they finish reading any other tile. Kitchen on the line glances at the screen with eyes already moving toward the next task — color does the triage work for them.
  • Thresholds, not gradients. Discrete tiers (fresh / aware / warn / late) beat continuous gradients because the human brain bins them — you don't need to read "11 vs 12 min", just "warn vs warn".
  • Pulse on late. Late tier adds a slow pulse so it draws the eye even on a glance. Aware/warn don't pulse — preserves attention for the actually-late tickets.

How it adapts to Cenas

  • Primary surface: EzCater pipeline. Today the cards probably show a created-at timestamp. Add an age field + a tier-classifier + the color treatment. Thresholds: 5/10/15 min make sense for catering confirm-back (Sam to validate against actual operations).
  • Secondary surface: Driver dispatch tiles. Same pattern, different thresholds — driver-not-acknowledged-yet age. Tile turns red if a driver hasn't accepted/started the run within X min of dispatch.
  • Tertiary: Produce-order-not-sent. If a manager opens an order and walks away, the cart sits. Age the cart so they see "12 minutes ago" in red when they come back.
  • Token additions: --age-fresh / --age-aware / --age-warn / --age-late map to existing semantic tokens (text-soft / gold / gold-bright / status-warn). No new colors needed — just the binding.
  • Animation discipline. Pulse on LATE only, 2.4s ease-in-out infinite — same cadence as the right-sidebar current-phase pulse (v3 mockup 298eeb8). Consistency across the app's animation vocabulary.

Implementation cost

Backend: trivial — compute age client-side from created_at. Frontend: ~20 lines CSS (tier classes + keyframes), ~5 lines JS to compute tier from age. Apply to existing tile templates. Total: ~half day for all three surfaces.

Citation

Toast KDS — see doc.toasttab.com/doc/platformguide/platformKDSOverview.html + Toast support docs on "Customize the Appearance of KDS Tickets". Pattern also in Square KDS, Lavu KDS, Olo Expo. Universal in restaurant ops; Cenas adopting it is table stakes, not differentiation.
