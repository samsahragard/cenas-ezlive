Source pattern

7shifts mobile home. The manager home shows a row of blocks summarizing staff state right now: On / On Break / Overdue / Upcoming / Out, each tappable to drill in. It answers "who do I have on the floor this minute" in one glance without opening the schedule. The five-bucket abstraction is the heart of the pattern — fewer buckets and you lose triage clarity; more and you lose at-a-glance scannability.

Why it works

  • Real-time, not "today's schedule". Schedules are abstract; "4 drivers are on a run RIGHT NOW" is concrete. Operators care about current state, not planned state.
  • Color-bound to action. Green = good (people working). Gold = warm (people idle or on break — opportunity to redirect). Red pulse = urgent (someone late). Gray = neutral (off-shift, no action needed).
  • Drill-down = same component, deeper. Clicking a bucket expands into the list of named individuals + their context. No screen change — overlay or expand inline.
  • Reusable abstraction. Same 5-bucket pattern works for drivers (On Run / Idle / Break / Late / Off), FOH staff (On Floor / Upcoming / Break / No-Show / Off), kitchen staff, vendors, etc. One CSS component, many surfaces.

How it adapts to Cenas

  • Primary v1 surface: per-store dashboard, drivers bucket. 5 blocks tied to driver_runs + driver_shifts tables. Late-detection: any run with arrived_at IS NULL AND expected_arrival_at < now() - 5min.
  • v2 surface: FOH/BOH staffing once you have a real schedule system. Same 5 bucket pattern, different data source.
  • Drill behavior: click a bucket → expand inline (don't navigate). For the Late bucket, the drill row should include 1-tap actions: "Ping driver" / "Reassign" / "Mark contacted." Otherwise it's just inspection, not workflow.
  • Don't double up with the needs-you strip (research #4). Strip says "X needs your action"; Right Now says "here's the team state." Complementary — Strip = TO-DO, Right Now = STATUS.
  • Late-block pulse only when count > 0. Pulse is annoying when always-on; valuable when it appears to draw attention.
  • Mobile responsive: 5-up grid collapses to 2-up on <720px (mockup demonstrates). Late + On stay first since they're the actionable buckets.

Implementation cost

Backend: aggregator query joining driver + driver_runs + driver_shifts (or current proxy tables). Returns {on, idle, break, late, off} counts + a few summary stats per bucket. ~half day. Frontend: ~half day for the 5-block grid component + drill-expand interaction. Late-detection threshold tuning: ongoing per Sam's input on what feels right. Total: ~1 day for v1 driver bucket; FOH adds when staffing data exists.
