Source pattern

Uber Eats Manager. A persistent location chooser at the top filters all metrics, with "All locations" as an aggregate option. The toggle lives in the app chrome (top bar), not inside any one page — so it survives navigation. Operators with 2+ locations spend a meaningful fraction of their time switching scope; making the switch one tap from anywhere is the whole game.

Why it works

  • Toggle in chrome > toggle in page. If the toggle is inside the dashboard page, the operator has to navigate back to switch scope. In the top bar, scope-switching is global and constant.
  • Default "Both" view = aggregate as first impression. Most operators want the big picture first ("how's the business today") then drill into a single store. Default scope = aggregate respects that pattern.
  • Per-store color identity. Tomball=blue, Copperfield=green — operators recognize scope at a glance from anywhere on the page (sub-numbers tinted, page accents shift).
  • Persists across navigation. Scope from /dashboard carries to /drivers carries to /orders. Operator picks scope once for a session, not once per page.

How it adapts to Cenas

  • Primary surface: base_dashboard.html topbar. 3-segment toggle: Tomball / Copperfield / Both. Lives globally — every partner-tier page that has store-scoped data inherits.
  • Routing model: today routes are /dos/... and /uno/... per-store. New unified routes /store/{scope}/... OR keep per-store URLs and add a global ?scope= override on those that support aggregate. Either works; aggregate-overlay is the cheaper path.
  • "Both" aggregate semantics: KPIs sum across stores. Order list interleaves with a per-row scope-dot indicator. Driver dispatch shows both stores' drivers on one map. Some pages (like a single store's pre-shift checklist) can hide the "Both" option since it doesn't make sense for that surface.
  • Per-store color tokens: add --ck-store-tomball: #6BA3FF + --ck-store-copperfield: #6FD87B. Used in toggle dots + page accent tint + sub-split numbers.
  • Persistence: URL query string for shareability + localStorage for default sticky-across-sessions. Combo: URL wins if present, otherwise fall back to localStorage, otherwise default "Both."
  • Sub-split numbers in aggregate view. When in Both scope, each KPI shows the aggregate big number + the per-store split underneath (small, tinted). Operator gets aggregate + composition in one card.

Implementation cost

Toggle component: ~30 lines (CSS+JS) wired into base_dashboard.html. Routing: depends on how store-scoped queries are structured today — if you have a clean get_orders_for_store(store_slug) helper, add store_slug == 'both' handling = trivial. If queries are baked into route handlers, this is more work (~2-3 days). Per-page sub-split display: ~per-page small touch (~half day per major page). Total: ~1-2 days for the chrome + ~1 day per major page that gains aggregate scope.
