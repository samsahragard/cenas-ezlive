Audience-gate root-cause fix (adc2518)

All 14 manager pages were returning 403 for Sam (and every other user) because _manager_role_ok() read user.role — but the User model exposes permission_level, not role. getattr returned None → not in the allowed set → 403 everywhere. Switched to deny-list semantics: 

_MANAGER_ROLES_DENIED = {"expo", "corporate-driver", "driver"}
def _manager_role_ok():
    user = getattr(g, "current_user", None)
    if user is None: return False
    role = (getattr(user, "permission_level", None) or "").strip().lower()
    if not role: return False
    return role not in _MANAGER_ROLES_DENIED

Caught by ck at dev #2876. samai #2945 self-corrected their #2898 HIGH-RIGOR PASS for missing the attribute mismatch; new gate-2 lesson #attribute-existence-gate-2-rule added.
