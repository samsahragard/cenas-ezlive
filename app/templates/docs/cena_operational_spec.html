Cena — Operational Spec

Reference doc for the Cena agent. Authored 2026-05-15 by aick at the end of Sam's PART 1-5 directive (the Cena tunnel rebuild + permission elevation). Read this before reverse-engineering anything; it captures the as-built configuration as of commit aa6074b.

1. What Cena is

Cena is Sam's personal operational AI surface. Distinct from:

  • aick / ck / samai — the dev-side Claude shells that build Cenas Kitchen. They develop the app; they do not run on live infrastructure.
  • The planned Block 3 manager-facing in-app agent — that surface is for restaurant managers (Tomball + Copperfield) and ships later. Cena is for Sam only.

Cena's job: take Sam's instructions on /sam/chat, execute them against the live infrastructure (codebase, Render, Cloudflare, Telegram), and report back. Every action is logged to /sam/cena-audit/ for Sam to review.

2. Access control

  • Gated to one user. SAM_CHAT_USER_ID env var on Render holds Sam's User.id. Both /sam/chat and /sam/cena-audit/ check g.current_user.id == SAM_CHAT_USER_ID at the route level — not via ROLE_PERMISSIONS, so the capability cannot be role-inherited.
  • No other user gets Cena access. Not Masood, not the partner tier, not managers. Cena is for Sam only.
  • Permission elevation is eyes-open. Sam authorised full unrestricted access on 2026-05-15 after the prompt-injection risk was flagged. On the record in the dev chat.

3. Tunnel topology

The path a Sam Chat message takes from the browser to Cena and back:

  1. Browser POST to /sam/chat/send on app.cenaskitchen.com (Render).
  2. app/web/sam_chat.py's gateway branch fires when CENA_GATEWAY_URL env var is set. It opens an httpx stream through the SOCKS5 proxy (CENA_PROXY=socks5h://localhost:1055) to http://100.108.119.19:8765/cena/stream.
  3. Render's userspace tailscaled (booted by start.sh) routes the connection through the tailnet (tailb5e6ee.ts.net, owner samsahragard@gmail.com) to AiCk.
  4. Cena's gateway (cena_gateway.py as a Windows scheduled task on AiCk, port 8765) authenticates the X-Cena-Token header, calls the Anthropic API with Cena's system prompt + tools, and streams the SSE response back.
  5. Each tool invocation along the way POSTs a CenaActionLog row to /sam/cena/log on Render (fire-and-forget thread).
  6. Sam Chat persists the final assistant turn with token usage and cost.

Architectural note (PART 3 lesson): the previous Cloudflare Tunnel + external-DNS CNAME path is broken by design when the parent domain isn't a Cloudflare zone (cfargotunnel hostname resolves only to a non-routable IPv6 ULA; bypassing DNS to a real CF anycast IP fails the TLS handshake because CF has no cert for the hostname). Tailscale-on-Render is the path we run; do not try to resurrect the CF tunnel without moving cenaskitchen.com nameservers to Cloudflare.

Adding a new tool means: define it in TOOLS, dispatch it in execute_tool(), and add the name to _VALID_CENA_ACTION_TYPES in app/models.py. The audit surface auto-renders new types; the validation set is forward-compatible (unknown types are logged but not rejected).

6. Audit log

  • Table: cena_action_logs on Render's SQLite at /var/data/cenas_kitchen.db. Migration 23_cena_action_log.py; model in app/models.py (CenaActionLog).
  • Ingest: POST /sam/cena/log. Auth: X-Cena-Token header (the same shared secret used between sam_chat and the gateway). Site-tier auth exempt (auth.py EXEMPT_PREFIXES).
  • View: GET /sam/cena-audit/. Sam-only. Filter UI by action, success/error, session, time window. 24h summary pills at top.
  • Links: each row optionally references sam_chat_sessions.id + sam_chat_messages.id — click-through is not wired in the UI yet but the data is there.

7. Escalate-on-uncertainty pattern

This is a behavioural pattern in Cena's system prompt, not a permission gate. Cena stops and calls telegram_send before executing when an action is uncertain. Concretely:

  • Destructive ops: git reset --hard, file_delete on shared paths, shell_execute with rm/del/format, render env-var deletion.
  • Pushing commits to main when the diff is large or Cena didn't write the code itself.
  • Changes that affect live-app availability: Render env-var changes touching auth, DB, or CENA_*; force-pushes.
  • API calls that move money (Stripe, Twilio billing endpoints).

Sam can override with "just do it" on a per-message basis. The default is to ask — the cost of a quick check is low; the cost of breaking the live app is high.

8. How Sam uses it

  1. Open /sam/chat (logged in via keypad as the SAM_CHAT_USER_ID match).
  2. Type a request in natural language. Sam doesn't need to know the tool names — Cena picks them.
  3. Watch the stream. Tool invocations are shown inline as [tool_name(args)] → preview; the assistant text follows.
  4. If Cena escalates with a Telegram ping, it currently appends to cena_telegram_outbox.log on AiCk — Sam reads from there until the real Telegram wiring is done.
  5. Review /sam/cena-audit/ any time to see what Cena actually did (parameters, results, errors, latency).

9. Operational gotchas (lessons from PART 3-4)

  • Tailscale userspace doesn't intercept syscalls. Must use the SOCKS5 proxy at localhost:1055 via httpx(proxy=...). Direct TCP connect to 100.x tailnet IPs from the Render container will timeout.
  • Single-use auth keys get consumed. Use Reusable + Ephemeral for the Render auth key so re-deploys can register new nodes. The key lives in RENDER_TS_AUTHKEY on Render.
  • UTF-8 BOM in secret files. Notepad on Windows can add a BOM. Read with encoding="utf-8-sig" on both sides of any shared secret. HTTP headers require latin-1 so a BOM in a header value fails on send.
  • Render's install.sh requires sudo. The build env runs as root but the installer doesn't detect that. Use the static tarball from pkgs.tailscale.com/stable/ instead.
  • tailscale.com stable static version ≠ Windows installer version. The JSON index at pkgs.tailscale.com/stable/?mode=json has the actual current static-channel version under TarballsVersion.
  • Site-tier auth intercepts unauthenticated POSTs. The Cena gateway has no site cookie. Endpoints the gateway needs to reach must be added to EXEMPT_PREFIXES in app/web/auth.py with their own token check inside.
