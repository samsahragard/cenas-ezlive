ezCater Review Queue retirement + auto-resolver

Sam wanted to delete the manual review queue. Final shape (after one iteration on his direction):

Pipeline

  1. Tier 1 — webhook ingest (existing): pull order via Partner API, validate, persist with needs_review = bool(all_warnings).
  2. Tier 2 — re-pull: if any warnings, wait 5s, call gql_pull(entity_id, _ez_token()) again, re-ingest. Often clears the warning because ezCater's backend was lagging at the submit-time read.
  3. Tier 3 — Claude triage: if warnings still remain, app/services/ezcater_resolver.py:try_claude_resolve() asks claude-haiku-4-5-20251001 whether each warning is a real problem or a false positive. Claude returns {"cleared": true|false, "notes": "..."}.
  4. If Claude clears it → _clear_order_review_flag(external_order_id) flips Order.needs_review = False. Order passes silently.
  5. If Claude can't clear it → needs_review stays True → order surfaces on the Partner-only queue.
  6. No Telegram on warnings (per Sam's later directive). Telegram still fires on success / hard ingest failure / unknown caterer.

Operating model with multiple Claudes

This session ran with two AI agents (aick + ck) coordinating via the Developer Chat. samai is announced but not yet active — Sam said at 3:10 PM that samai will join with aick coordinating both ck and samai going forward. Patterns we settled on with two-agent traffic (apply equally once samai is on):

  • Pull-rebase before commit; only commit your own files (git add by name, not -A) to avoid sweeping in another Claude's in-progress edits.
  • SSH from CK can't push due to wincredman — only AiCk's desktop session has the GitHub creds. So ck commits locally on AiCk via SSH, then asks aick on chat to push. Cumbersome but stable. (samai will hit the same wall if they're remote.)
  • Coordinate touchpoints in chat first: name the files you're about to edit before you start so the other Claude can stay clear.
  • Echo your own posts — the Monitor stream replays your own messages back. Don't re-respond to them.
  • Sam's spec changes mid-build — be prepared to revert. The sidebar UX iterated four times in one morning. Don't get attached to the prior implementation; ask if a constraint was relaxed before assuming.
