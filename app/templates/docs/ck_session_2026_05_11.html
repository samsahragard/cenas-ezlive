Toast cache: Sunday-zero bug + labor donut $0 (commits 5a14d4e + be1d7e1)

Sam (correctly) called out that the partner sales report showed $0 for Sunday 5/10 even though the restaurant was open and busy. Same shape an hour later: dashboard Labor donut showed $0 / "no labor recorded yet" with 23 shifts logged.

  • Root cause: Toast caches on disk under /var/data/toast/. fetch_orders_for_date and fetch_time_entries both wrote orders_YYYYMMDD_<loc>.json / timeentries_…_<loc>.json and read forever from the first cached result. When an early-morning probe hit Toast before sales were finalized for the day, the cache stuck at empty for the rest of the day.
  • Fix pattern (applied to both fetches): for dates within the last 2 calendar days, only trust the cache if it has data AND is <30 min old. Otherwise re-fetch from Toast. Past dates (>2 days old) still cache indefinitely since Toast doesn't backfill.
  • Verified post-deploy: sales = $9,048 across 4 channels, labor = $1,800 / 169.1 hrs / 62 shifts / BOH+FOH split. Donuts populate on first dashboard load now.
  • Two other Toast cache files (employees_<loc>.json, jobs_<loc>.json) deliberately untouched — these are slowly-changing rosters/titles where forever-caching is correct.

Produce ingest pipeline deep-dive

Sam: "the prices aren't updated on the app." Took most of the afternoon to root-cause; turned out to be three layered bugs.

Bug 1: stale unfiltered "items not available" wording (commit ce48adf)

Old summary text was "X canonical items not currently available from either vendor." Sam: produce items are always available — what's missing is just our current quote. Reworded to "X items missing from the latest vendor price lists — still always available, we just don't have a current quote."

Bug 2: missing ANTHROPIC_API_KEY (env-var fix via Render API)

Alvarado emails arrive as JPG images and parse through the Anthropic Vision API (claude-sonnet-4-6) in produce_ingest.extract_from_image. Render's ANTHROPIC_API_KEY env var was unset, so every Alvarado parse threw RuntimeError: missing ANTHROPIC_API_KEY and the email got marked processed with an empty result.

  • Set the key via Render API (PUT /v1/services/<id>/env-vars/ANTHROPIC_API_KEY); key sourced from ~/.openclaw/.secrets/anthropic_api_key.txt on AiCk. Auto-triggered a redeploy.
  • After the deploy + a forced re-ingest, Alvarado's 5/10–5/16 price list parsed successfully (176 line items extracted).

Bug 3: aliases.json drift + incomplete coverage (commits eaaf493 + f9c0eb0)

_apply_aliases in produce_ingest.py did exact-string match between vendor item names + aliases.json keys. Vendor strings like 'AGUACATE 32 CT # 1|32CT' didn't match aliases like 'AGUACATE 32CT #1|32CT' because of trivial whitespace drift (likely Claude Vision output evolving). And the JLuna aliases section only had 14 entries when the vendor sends 114 — many of Sam's canonical items had no bridge at all.

  • eaaf493: added _normalize_alias_key that collapses whitespace and joins digit-unit pairs (32 CT → 32CT, # 1 → #1) before match. Both vendor strings and alias keys pass through the normalizer.
  • f9c0eb0: 27 new JLuna alias entries bridging Sam's canonical items (Cabbage Red Ctn, Carrots Jumbo, Eggs Grade AA XL, Ginger, Lett. Iceberg/Romaine/Green Leaf, Lemons 200ct + 140ct, Mangos, Mushroom Gen/Cello, Onions Yel/White/Red Jbo, Oranges Calif/Cal, Pepper Green Bell/Jalapeno/ Poblano/Serrano, Potatoes A Red/B Red, Squash Mexican/Zuchinni, Tomates Roma, Tomatillo Pelon).
  • Added a WARNING log when >30% of a vendor's items stay unmapped after normalization — flags alias drift in logs immediately next time a vendor changes their format.
  • Aick was going to do the Alvarado-side new aliases in parallel but it turned out unnecessary: my JLuna entries plus the normalization fix gave each of Sam's 38 canonical items at least one priced match (Alvarado wins on price for Spanish- named items, JLuna wins on the English-named ones).

Forced re-ingest (transient diagnostic endpoint)

To verify the fix without waiting for the next vendor email, I shipped a transient admin endpoint at /produce/admin/ingest-state gated by the INGEST_TOKEN bearer (same token aick uses for ingest endpoints). Supports:

  • GET — dumps ingest_state.json + approved_senders.json.
  • ?dump_vendor=alvarado|jluna|canonical — inspects the live vendor JSON files + canonical items list.
  • ?reset_last_seen=N&clear_processed=1 — bumps the poller's last_seen_mid back so it re-evaluates past emails.
  • ?patch_vendor=jluna&set_date_range=… — manually overrides a vendor's date_range string. Used to mark JLuna's stale 5/8–5/9 list as effective 5/10–5/16 per Sam's "just for this time" directive.

All four pieces are transient — should be reverted before the next deploy round (or kept as a real admin tool if Sam finds it useful). Routing exemption in EXEMPT_PREFIXES covers /produce/admin/.
