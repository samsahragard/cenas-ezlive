Platform: SECRET_KEY rotation on Render

Sam asked to log out everyone in the app. Discovered that Render didn't have a SECRET_KEY env var set at all — production was falling back to the 'dev-secret' string from app/__init__.py line 39. Rotated by creating a fresh 64-byte secret via the Render API (PUT /v1/services/srv-d7ue6ivlk1mc73aaka20/env-vars/SECRET_KEY). Every Flask session cookie (keypad sessions, driver sessions, anonymous tier-1 site-password sessions) is now invalidated. Side effect: closed a real security hole — anyone who knew the 'dev-secret' fallback could have forged Flask cookies against production.
