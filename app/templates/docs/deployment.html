Deployment overview

Render.com auto-deploys on every push to the main branch of github.com/leverr18/cenas-ezlive. Build takes ~2-3 min; pre-deploy + spin-up adds another ~2 min, so total time from git push to live is typically 4-5 min. The 502 page during deploy swap is normal and resolves in ~30s.

Persistent disk

1 GB at /var/data. Holds:

  • cenas_kitchen.db — the SQLite DB. Path is wired in via the DATABASE_URL env var.
  • toast/* — Toast API cache files (orders, employees, jobs, time entries per location).
  • sling/* — Sling API cache (groups, users, calendar windows).
  • produce/* — vendor JSONs (current week's parsed prices) + ingest state + IMAP attachment files.

DB schema management

Two paths exist; the alembic path is the right long-term answer but currently isn't wired:

  • Alembic migrations in migrations/versions/. Runs via alembic upgrade head when configured as the Render preDeployCommand. Currently the field is unset.
  • Backstop: create_app() calls Base.metadata.create_all() on startup. This creates any new tables that aren't on the live DB but does NOT apply ALTERs to existing tables. Safe for adding entirely new model classes; not safe for adding columns to existing models.

If you add a new column to an existing model: add a migration AND consider a code-level idempotent op.add_column(if_not_exists=True) until preDeployCommand is wired.

Common deploy tasks

Trigger a deploy without a git push

Push to main auto-deploys. To trigger without code changes (e.g., to pick up env-var changes), use the Render dashboard's "Manual Deploy" button OR call the Render API:

POST https://api.render.com/v1/services/srv-d7ue6ivlk1mc73aaka20/deploys
Body: {"clearCache":"do_not_clear"}
Auth: Bearer <render_api_key>

Set or update an env var

PUT https://api.render.com/v1/services/srv-d7ue6ivlk1mc73aaka20/env-vars/<NAME>
Body: {"value":"<new value>"}

(See the secrets path doc for where the Render API key is stored.)

Tail logs

GET https://api.render.com/v1/logs?ownerId=<owner>&resource=srv-d7ue6ivlk1mc73aaka20&limit=100&direction=backward

Local development

  1. Clone github.com/leverr18/cenas-ezlive to C:\Users\sam\Desktop\cenas-kitchen-claude\
  2. Create venv: python -m venv .venv
  3. Install deps: ./.venv/Scripts/pip install -r requirements.txt
  4. Set env vars in a local .env (the app's db.py calls load_dotenv)
  5. Run dev server: ./.venv/Scripts/waitress-serve.exe --listen=127.0.0.1:5000 wsgi:app or flask run
