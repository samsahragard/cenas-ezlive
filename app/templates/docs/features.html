Sidebar tour

Every feature on the site is reachable from the sidebar inside a store context. Below: what each leaf does, where the data comes from, and any privacy / role behavior.

TODAY → [Store] Dashboard

Manager landing page. KPI strip (today's deliveries, heads to feed, orders needing review, produce winners), today's per-delivery list with status badge, "needs attention" panel, and section cards. Filters automatically to the current store's location.

OPERATIONS → Tomball / Copperfield (per-store sidebar)

Routes through to the existing /orders/<location> view — list of catering orders for that location with rolling date filter (today + upcoming).

OPERATIONS → Vendors → Produce → Orders

Per-item price history page. KPI strip (items tracked, biggest movers, snapshot window). Biggest-movers callout flags items where the most recent two snapshots per vendor moved ≥ 5% (red ↑ / green ↓). Per-item Chart.js line chart with separate Alvarado + J. Luna lines, lookback dropdown 30 / 60 / 90 / 180 / 365 days. Latest-pricing table with cheaper-vendor highlighted in green.

Data: produce_price_snapshot table, populated by produce_ingest.py every time the IMAP poller parses a fresh vendor email.

OPERATIONS → Vendors → Produce → Order Guide

The original produce ordering UI — daily list of items with cheaper-vendor pricing + qty input, single submit. Sends an order email to the vendor + a Telegram confirm link to Sam.

OPERATIONS → Ezcater → orders / order processor / review queue

  • orders: per-location catering order list (Tomball or Copperfield based on URL prefix), today + upcoming, status filter excludes cancelled.
  • order processor: PDF upload page — uses Anthropic vision to extract orders from PDFs (legacy fallback for cases where the ezCater webhook misses).
  • review queue: orders flagged for review (extraction confidence warnings, missing customer name, unparsed dates, etc.).

OPERATIONS → Ezcater → Driver Tracking

Renamed from "Manager Dashboard". Per-location managers log driver performance (mileage, on-time, tracking, photo, 5-star, notes). Reads from the driver_logs table.

OPERATIONS → Ezcater → Driver Portal

Driver-facing view. Currently shows assignments and delivery details. (Future: per-driver login + GPS tracking — not yet built.)

OPERATIONS → Schedule → BOH / FOH / All Roster

Three filtered views of the Sling-sourced employee roster. BOH = at least one BOH position. FOH = at least one FOH AND no BOH. All = everyone. Position pills show every role each person can fill. "Toast" badge surfaces when an employee has a linked Toast Payroll record.

Data source: Sling /groups/<id>/users for each location group, cross-referenced with each position group's members. BOH/FOH classification is in app/services/role_classifier.py.

OPERATIONS → Schedule → Weekly

Sling-sourced shift schedule for a date range. KPI strip (total shifts, hours, open shifts), per-day shift tables, by-position rollup. Quick-pick buttons: This week / Next 2 weeks / Next 4 weeks / Past week. Range capped at 28 days. Covers BOTH locations — Toast's API only mirrors one location's shifts, so we hit Sling directly for full coverage.

INSIGHTS → Performance → Server / Bartenders / All

Per-server tip percentage + service timing (avg drink, apps, entree, drink→entree gap, ticket duration, CC tabs/tips, tip %). Filters to the requested role:

  • Server: Server + Server Trainee + Lead Server
  • Bartenders: Bartender
  • All: Server + Trainee + Bartender + Host

Data: Toast /orders/v2/ordersBulk per business date, parsed for selection timing + payment data. Items classified into drink / appetizer / entree using a 1,200-item bundled lookup with keyword fallback.

INSIGHTS → Sales → Toast / Online / DoorDash / Uber / Total / EzCater

Per-channel sales rollup. Each leaf filters Toast orders by channel:

  • Toast: dine-in (source = "In Store")
  • Online: Toast Online Ordering (source = "Online")
  • DoorDash: orders with source = "API" + DoorDash placeholder address ("1 DoorDash Value", San Francisco)
  • Uber: similar API-channel detection (currently empty — no Uber Eats orders yet)
  • Total: all channels combined including in-store
  • EzCater: placeholder — orders are in the Order DB but per-item pricing isn't yet captured at the order level

INSIGHTS → Labor → BOH / FOH

Labor cost by Toast job title with % of net sales (industry-standard labor cost ratio). Splits BOH (kitchen — Cook / Prep / Grill / Dish / Enchilada / Kitchen Mgr) and FOH (Server / Bartender / Host / Cashier / Busser / Expo / Floor Mgr).

Privacy: management positions (Kitchen Manager, Floor Manager, General Manager, Owner, Corporate, Asst Kitchen Manager) show only their % of net sales — not people count, not hours, not labor cost — in Tomball / Copperfield / Corporate views. Only the Partner view shows full management detail.

DEVELOPER → Chat (Partner only)

Persistent thread for Sam + this Claude + future AI agents. Auto-polls every 4s. Author dropdown supports sam / aick-claude / ck-claude / system / custom. Color-coded by author.

DEVELOPER → App (Partner only)

These docs (you're reading them now). Read-only. Updates only via Sam-approved git commits — no edit UI.
