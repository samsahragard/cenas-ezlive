Cena tunnel: known architectural limitation

During the 2026-05-14 session, empirical testing established that Cloudflare Tunnel via an external-DNS CNAME to cfargotunnel.com does NOT route end-to-end when the domain isn't a Cloudflare zone:

  • 259354d8-…cfargotunnel.com resolves to fd10:aec2:5dae:: (an IPv6 ULA, non-routable, no IPv4 A record)
  • Bypassing DNS with curl --resolve to a real Cloudflare anycast IP fails the TLS handshake (SEC_E_ILLEGAL_MESSAGE) — Cloudflare's edge has no cert for a hostname outside its zones
  • Conclusion: Cloudflare Tunnel + custom domain only works when the domain is on Cloudflare's DNS (or Cloudflare for SaaS / Enterprise). Sam ruled out moving DNS to Cloudflare ("no nameserver change").

Open architectural question (to be picked up tomorrow): given the constraint, options for the Render → Cena path are:

  1. Tailscale on Render (userspace tailscaled in the Render service) — Cena stays tailnet-only, Render joins the tailnet and reaches http://100.108.119.19:8765 privately. CENA_GATEWAY_URL becomes the tailnet IP. No pretty cena.cenaskitchen.com URL.
  2. Render-proxies-to-Cena via Tailscale — Render runs Tailscale userspace AND the Flask app gets a /cena/* reverse-proxy route. cena.cenaskitchen.com becomes a Render custom-domain CNAME, Render handles TLS. Most setup; gets the pretty URL.
  3. Cena stays tailnet-direct, Render integration deferred — only Sam/ck/aick use Cena directly from tailnet machines. ck's a32b691 Cena routing stays dormant.

Tailscale itself is already installed and running on AiCk (verified via tailscale status: aick 100.108.119.19, ck 100.73.38.82, sam 100.79.36.67). However, cena_gateway.py currently binds host="127.0.0.1" — localhost only. Other tailnet devices cannot reach 100.108.119.19:8765 yet. Changing the bind to host="0.0.0.0" is a one-line prerequisite for any tailnet path.
