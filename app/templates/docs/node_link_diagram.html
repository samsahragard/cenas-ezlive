What this page is

Interactive node-link diagram of the whole system. Each node is an entity (a user, an auth tier, a page, an external service, a database table, a background worker). Edges show how things connect — who logs in where, what page calls which service, which service reads or writes which table. Drag any node to rearrange. Hover to see details. Filter with the group buttons below.

How to read: nodes color-coded by group. Edges are arrows showing the direction of the relationship — "user → tier" means the user must pass that tier; "page → service" means the page calls that service; "service → table" means the service writes (or reads) that table. Hover any node to see what it does. Drag to rearrange. Use the filter buttons to focus on a single layer.

How to use this diagram

  • Hover a node — its description + every in/out edge appears in the panel below the canvas.
  • Click a node to pin its detail; click anywhere else to clear.
  • Drag a node to move it; release and physics re-stabilizes.
  • Filter buttons trim the graph to a single layer (Users + Auth / Services + Data) or a single flow (Driver path, Manager path, ezCater flow, Payroll flow). "All layers" restores the full graph.
  • ↻ Re-layout re-runs the physics simulation. ⤢ Fit centers the graph in view.
