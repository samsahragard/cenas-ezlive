What this page is

End-of-day routine for every Claude on the project (aick, ck, samai). Each of you does all six steps in order before Sam restarts the session — so the next instance picking up cold has the full picture and the conversation transcript is archived locally.

1) Write your session log

File: app/templates/docs/<your-name>_session_<YYYY>_<MM>_<DD>.html  (e.g. ck_session_2026_05_12.html, aick_session_2026_05_12.html, samai_session_2026_05_12.html)

Format: copy ck_session_2026_05_11.html as your template.

Required sections in this order:

  • "What this page is" — one paragraph + day-in-two-sentences
  • "Session bootstrap" — how you re-attached, monitor setup, naming
  • One section per major thing you shipped, with commit hashes
  • "Live-tested" — what you actually verified end-to-end
  • "Commit chain (your pushes)" — list with one-line summaries
  • "Open threads for the next <your-name>" — what's unfinished or about to land. Specific. Files + line numbers if possible.
  • "Cross-claude coordination notes" — what you handed back/forth with the other claudes

2) Register your log under the Chats folder

Edit app/web/developer_chat.py — append a tuple to CHAT_PAGES (not DOC_PAGES — chats live in their own folder so the top doc nav stays short; see commit 922f0f5): 

("<your-name>-session-<YYYY>-<MM>-<DD>",  "<your-name> — M/D",
 "doc_<your_name>_session_<YYYY>_<MM>_<DD>"),

The log will then show as a card on /partner/developer/app/chats and a direct deep-link at /partner/developer/app/<your-name>-session-<date> keeps working.

3) Copy your conversation jsonl

Source: C:\Users\sam\.claude\projects\C--Users-sam\<your-session-uuid>.jsonl

Destination: C:\Users\sam\Desktop\claude conversations\conversations\<YYYY>-<MM>-<DD>-<your-name>.jsonl

e.g. 2026-05-12-aick.jsonl, 2026-05-12-ck.jsonl, 2026-05-12-samai.jsonl

If your session is still active find the uuid via:

ls -lat C:\Users\sam\.claude\projects\C--Users-sam\*.jsonl

The most-recent jsonl is yours.

4) Git — stage + commit + push

  • CK and SAMAI: commit locally (do NOT push — you don't have remote creds). Tell aick the sha in chat and ask "aick please push <sha>".
  • AICK: pull, then push. After CK/SAMAI ping you, push their commits too.

Commit message format:

docs: <name> session <YYYY>-<MM>-<DD> handoff log

<one-paragraph summary>

Reachable at /partner/developer/app/<name>-session-<YYYY>-<MM>-<DD>.

5) Post to chat

<name> session-restart prep (<sha> deploying):
  1) Session log: /partner/developer/app/<name>-session-<date>
  2) Conversation jsonl: copied to Desktop\claude conversations\conversations\<date>-<name>.jsonl
  3) Open threads: <one-line summary of biggest thing still in flight>

Ready for session restart whenever you give the word.

6) Stop your background tasks (only if Sam tells you to)

  • Kill any Monitor running chat_tail.
  • Kill any local http.server / dev server you started.
  • Tell Sam which task ids you stopped.

Checklist for Sam to verify (before saying "restart")

  • ☐ Three new files in app/templates/docs/ (one per AI)
  • ☐ Three new CHAT_PAGES entries
  • ☐ Three new files in Desktop\claude conversations\conversations\
  • ☐ git log shows 3 new commits with your-name session handoff
  • ☐ Render shows the latest deploy as live
  • ☐ /partner/developer/app/<name>-session-<date> renders for each
