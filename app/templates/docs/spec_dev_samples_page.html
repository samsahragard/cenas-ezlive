Spec — Developer · Samples page

Spec authored by dck (2026-05-17) per cena #2420 + #2426 green-light. Implementation by ck; samai canonical-three-gate per Rule-4 on ship.

1 · Purpose

A simple, scannable list of dck's design samples + key V2-system reference implementations. Replaces the "post raw mockup links in chat" anti-pattern surfaced by Sam #2418. Lives under the Developer section in the sidebar, slot directly below Chat. dck's home for all future mockup deliverables.

2 · Route + registration

  • URL: /partner/developer/samples
  • View function: dev_chat.samples in app/web/developer_chat.py (fits the existing developer-section module — same lane as app_doc)
  • Active key: doc_dev_samples (matches doc-page key pattern)
  • Permission: developer.view_app_docs — same gate as existing dev docs
  • Partner enforcement: reuse _enforce_partner() at route entry, same as other developer-section routes

3 · Data source

v1: module-level constant in a small new file. No DB. Add a sample = one dict append per commit. DB-backed only if/when we need authoring through the app rather than through commits. 

# app/web/dev_samples.py
DEV_SAMPLES = [
    {
        "slug": "drivers-redesign-v2",
        "title": "Drivers (Admin) — Redesign Mockup",
        "version": "v2",
        "date": "2026-05-17",
        "author": "dck",
        "blurb": "Active/Inactive tabs, 48×48 hamburger in topbar "
                 "(closes title-collision class), tappability affordance "
                 "with chevron+hover. Desktop + phone.",
        "url": "/static/mockups/drivers_redesign.html",
        "type": "mockup",          # "mockup" | "reference"
        "button_label": "Open mockup",
    },
    # ... see §8 for v1 seed entries
]

Type field (added per cena #2451 finding 2): distinguishes sandbox mockups ("mockup" — static HTML in /static/mockups/, safe to click through) from live pattern references ("reference" — shipped production pages used as V2 design system examples; operators clicking through land in real data). The type drives both the tag pill (§6.2) and the per-entry button label (§6.4).

button_label: per-entry primary CTA text. Computed in template if not stored (simple if-else on type). "Open mockup" for type=mockup, "Open page" for type=reference. Extensible if future types need different verbs.

Pass samples=DEV_SAMPLES into the template render. Most-recent-first is sort-by-date desc — module-level can be hand-ordered for v1, swap to sorted(DEV_SAMPLES, key=lambda s: s['date'], reverse=True) when needed.

5 · Template structure

  • Path: app/templates/dev_samples.html
  • Extends: base_dashboard.html (universal chrome — topbar, sidebar, ribbon)
  • Page title: "Design Samples"
  • Subtitle: "Design mockups, V2 reference implementations, and structural samples. Authored by dck. — Most recent first."
  • Content: single container .dvs-list (vertical stack), one .dvs-card per sample, optional .dvs-empty state.

6 · Visual design (V2 tokens only)

6.1 · Card

  • Background: rgba(40, 22, 14, 0.55) — V2 glass-card base, same as .lg-card + .ezm-order
  • Border: 0.5px solid var(--ck-gold-soft)
  • Border-radius: 12px
  • Padding: 18px 22px desktop, 14px 16px mobile
  • Hover (desktop): border-color → var(--ck-gold), bg deepens to rgba(40, 22, 14, 0.72), transition 200ms var(--ck-ease)
  • Gap between cards: 14px desktop, 12px mobile
  • Max-width: 920px (read-friendly column width)

6.2 · Card head (title row)

  • Layout: flex row, align-items baseline, gap 12px, margin-bottom 6px
  • Title (<h2>): 16px / 600 / var(--ck-text)
  • Version badge: 10px / 700 / uppercase / letter-spacing 0.06em / bg var(--ck-gold) / color #140A06 (dark on gold) / padding 2px 8px / radius 4px. Render only when s.version is present and not '—'. Jinja conditional: {% if s.version and s.version != '—' %}...{% endif %}
  • Type tag pill (added per cena #2451 finding 2): 10px / 700 / uppercase / letter-spacing 0.06em / padding 2px 8px / radius 4px. Rendered for every entry — semantic signal about "what kind of thing this is." Two variants:
    • .dvs-tag-mockup — bg var(--ck-gold-soft), color var(--ck-gold), border 0.5px solid var(--ck-gold-soft). Renders as label "MOCKUP".
    • .dvs-tag-reference — bg rgba(232, 122, 62, 0.18), color rgba(245, 180, 142, 1), border 0.5px solid rgba(232, 122, 62, 0.3). Same orange-pill family as .location-pill in drivers_redesign.html. Renders as label "REFERENCE".
    Both palette-aligned (gold + orange brand family). No teal — keeps the same brand discipline samai #2018 called for.
  • Meta (author · date): margin-left auto (right-aligned) / 11px / uppercase / letter-spacing 0.1em / var(--ck-text-soft)

6.3 · Blurb

  • 13px / var(--ck-text-muted) / line-height 1.55
  • Margin: 6px 0 14px

6.4 · Card foot (action row)

  • Layout: flex row, align-items center, gap 10px
  • Primary "Open" button (.dvs-btn-open):
    • bg var(--ck-gold) / color #140A06 (dark on gold)
    • padding 8px 14px, radius 8px
    • 13px / 600
    • icon ti ti-external-link + per-entry label from s.button_label ("Open mockup" for type=mockup, "Open page" for type=reference) with 6px gap. Simple template fallback if not stored: {{ s.button_label or ('Open mockup' if s.type == 'mockup' else 'Open page') }}
    • target="_blank" rel="noopener"
    • hover: filter: brightness(1.08), 150ms var(--ck-ease)
  • Secondary link affordance (.dvs-link-secondary):
    • color var(--ck-text-soft), 12px, no underline
    • icon ti ti-link + the URL text (small, for copy)
    • hover: color → var(--ck-gold-bright)
    • title attribute "Copy link" for hover-tooltip affordance

6.5 · Empty state

  • Centered, 60px top/bottom padding
  • Icon ti ti-photo-off, 32px, opacity 0.4
  • Copy: "No design samples yet" — var(--ck-text-faint), 13px

7 · States

  • Default: card visible, all info present
  • Hover (desktop): per §6.1
  • Open button hover: per §6.4
  • Secondary link hover: per §6.4
  • Empty: per §6.5
  • No version badge: when version is None or '—', skip the badge entirely (don't render an empty placeholder)

9 · Mobile-vs-desktop

  • Always single-column list (no card grid). The 920px max-width is the only horizontal constraint.
  • At ≤720px: card padding shrinks 18px→14px / 22px→16px. Card-head wraps; meta drops to a second line (becomes full-width below the title/badge).
  • No surface-specific 1024px breakpoint needed — universal chrome handles the sidebar drawer, content adapts inside.

10 · Accessibility

  • Each card title is an <h2> for proper heading hierarchy below the page <h1>.
  • "Open" link uses target="_blank" rel="noopener" (security: prevents tabnabbing).
  • Secondary link has title attribute for hover-tooltip affordance.
  • Touch targets: "Open" button is ~32px tall via current padding. If on-device testing on small phone screens shows it as too small, bump padding to 10px vertical (yields ~40px tall, still under the 48px touch standard but acceptable for inline action). Real primary touch target is the whole card if we wire row-click — defer that to v2.
  • Empty state icon has decorative role only; the copy ("No design samples yet") is the accessible text.
  • Color contrast: var(--ck-text) on rgba(40, 22, 14, 0.55) over the dark page bg = high-contrast (WCAG AA pass). var(--ck-text-soft) for meta line is borderline — verify in browser before final ship.

11 · Animation

  • Card hover transitions: 200ms var(--ck-ease) (cubic-bezier(0.22, 0.8, 0.24, 1) brand motion)
  • Open-button hover: 150ms var(--ck-ease) on filter
  • No entrance animations on the list (avoid noise on every page load).

12 · Implementation notes for ck

  • One commit recommended: route + dev_samples.py module + template + sidebar entry + v1 seed data, all together. Atomic — easier samai-gate diff review.
  • No new CSS module file needed — inline <style> in the template is fine for v1 (matches the pattern of sam_chat.html, ez_manage.html). If the same .dvs-* patterns get used elsewhere later, extract then.
  • Tabler icons used: ti-photo (sidebar), ti-external-link (Open button), ti-link (secondary), ti-photo-off (empty state). All in the existing webfont — no CDN change.
  • Don't carry the mockup's .is-hover-demo trick from drivers_redesign.html into this page — that was mockup-only static-render demonstration. Production uses only :hover (per cena #2425 spec note).
