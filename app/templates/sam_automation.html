Cross-surface auto-mirrors

Server-side fan-out so a single Cena reply lands on every surface Sam might be reading from without Cena having to call a tool per surface.

wake-on-hub → /sam/chat Gateway helper Live
What it's for
Every Cena reply on the LAN hub is also auto-posted to the active /sam/chat session on Render. Sam sees the same message in both places without Cena thinking about it.
What runs it
_mirror_to_render_sam_chat() in cena_gateway.py, fired at the end of every wake-on-hub turn.
wake-on-hub → cena_sam_chat Gateway helper Live
What it's for
Every Cena reply on the LAN hub also lands in cena_sam_chat as a native author='cena' message — the proper Cena lane, not the dim "dev-mirror" bubble.
What runs it
_post_to_sam_chat("cena", ...) in cena_gateway.py.
bridge: dev chat → cena_sam_chat bridge.py helper Live
What it's for
Every non-cena dev-chat message is mirrored into cena_sam_chat with a dev-<author> prefix so Sam sees one unified stream. Cena posts are skipped here because the gateway writes those directly (no double-print).
What runs it
_mirror_dev_to_cena_sam_chat() in bridge.py.
/sam/chat → cena_sam_chat Flask helper Live
What it's for
Sam's posts on /sam/chat and Cena's replies there both auto-mirror to cena_sam_chat as sam-online / cena-online bubbles (gold border tag). Display-only — no wake re-trigger.
What runs it
_mirror_to_cena_sam_chat() in app/web/sam_chat.py, fires after persist in the send + persist-assistant paths.
Auth
CENA_SAM_CHAT_TOKEN env var, routed via CENA_PROXY (Render's userspace Tailscaled).
cena disconnected from dev chat Two-layer block Active

Per Sam directive 2026-05-23: cena does not appear in /partner/developer/chat. (1) Bridge skips author='cena' on PUSH. (2) Render /partner/developer/chat/post rejects author='cena' server-side regardless of source.

Mobile build + deploy pipeline

Capacitor 7 wraps the live web app at app.cenaskitchen.com in a native Android WebView. GitHub Actions builds the signed bundle; scripts/upload_aab.py pushes it to Play Console without anyone touching the Console UI.

mobile-android.yml (GitHub Actions) Build + sign + upload Live
What it's for
End-to-end Android build. Gradle assembleRelease produces the signed .aab; then scripts/upload_aab.py uploads it to the Play Console internal testing track so Sam can update the app on his phone without drag-and-dropping the bundle in Play Console.
Trigger
Push to origin/main that touches the mobile/ tree (or manual workflow dispatch).
Auth (signing)
Signing keystore + password supplied as GitHub Actions secrets.
Auth (upload)
GOOGLE_PLAY_PUBLISHER_JSON secret — the service-account JSON for ck-play-publisher@favorable-valor-493419-r9.iam.gserviceaccount.com (granted App permissions on com.cenaskitchen.app, 13 perms incl. "Release apps to testing tracks").
scripts/upload_aab.py Android Publisher API client Live
What it's for
Uploads a signed .aab to Play Console via the Android Publisher v3 API. Drives the full Edit → Upload → Track → Commit flow with cleanup on failure. Default track internal; --dry-run creates the edit + uploads the bundle but doesn't commit (sanity-checks against a non-prod track).
Trigger
Called by mobile-android.yml after gradle assembleRelease, or manually for one-off uploads.
Auth
Service-account JSON. Default path: ~/.openclaw/.secrets/play_publisher.json (ACL-locked). Env override: GOOGLE_PLAY_PUBLISHER_JSON_PATH or GOOGLE_PLAY_PUBLISHER_JSON (raw JSON for CI).
Exit codes
0 success · 1 bad input · 2 auth failure · 3 upload failed · 4 commit failed.
Capacitor App.backButton listener base_dashboard.html · Layer 3 Live

Falls through to default behavior if the Capacitor bridge isn't injected (i.e. when loaded as a browser tab).
Stack
