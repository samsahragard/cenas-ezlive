Architecture

High-level system design + the operational map of every account, machine, and data source.

System at a glance

The platform is a Flask application served from a Render web service, sitting behind a Cloudflare-fronted partner gate, with a Postgres database, scheduled cron services for background work, and several outbound integrations.

  • Web app — app.cenaskitchen.com, Render-hosted Flask + Jinja templates.
  • Database — SQLite on Render's persistent disk at /var/data/cenas_kitchen.db; SQLAlchemy ORM + boot-time create_all for migrations (Alembic not wired on Render).
  • Background jobs — Render cron jobs (Toast refreshes, anomaly briefs, calendar rolls, event-traffic refresh, sales insights, etc.).
  • External integrations — Toast (POS + analytics), Sling (scheduling), EZCater (catering intake; webhook + IMAP + Selenium UI flow), vendor email channels, Sentry (error monitoring).
  • AI surface — Cena gateway (HTTP + Tailscale) on AiCk; /sam/chat talks to it for the Sam-facing assistant. Anthropic + Gemini are the model providers.
  • Mobile shell — Capacitor WebView app com.cenaskitchen.app ("Cenas Kitchen Employee", Play Console internal testing) that loads app.cenaskitchen.com directly; built + signed in GitHub Actions, uploaded via the Android Publisher API.
  • Driver-assignment Selenium flow — webhook auto-assigns + manager dropdown re-assigns drive the partnerportal.ezcater.com UI via Selenium (aick) / Playwright (ck failover) on internal hosts. Render only creates job rows + POSTs wakes; never touches Chrome itself.

Operational inventory

Accounts (top-level)

  • Render — hosts the web service + crons + Postgres.
  • Cloudflare — DNS + Tunnel for internal-service exposure.
  • Tailscale — VPN mesh across machines.
  • GitHub — source control; SSH deploy key authorizes pushes.
  • Anthropic — API key; primary AI provider.
  • Google — Gemini API + Workspace.
  • Toast — POS account + Analytics OAuth app.
  • Sling — scheduling.
  • EZCater — catering intake.
  • Sentry — errors.
  • 1Password — canonical credential vault (CLI integration available).

Machines

  • Render web service — production app + cron runtime. SQLite lives on the Render persistent disk; restarting the service does NOT wipe data.
  • AiCk (Mini_IT12, 100.108.119.19) — primary internal host. Runs the Cena gateway (port 8765, exposed via Cloudflare Tunnel), the Selenium driver-assignment worker, the LAN-hub Telegram bridge, and the persistent /sam/chat ↔ Cena local chat (port 8770, Tailscale-only).
  • Mini_IT13 (ck's box, 100.73.38.82) — peer internal host. Runs the LAN team chat hub (port 8765, hub.py), the cena2 gateway (port 8766) with the Playwright failover for /jobs/driver-assign, and cross-gateway message queue sync.
  • Operator phone — Tailscale member; reads Telegram via @Cenasbot for dev-chat traffic.

Credential locations

The Pass tab is the source of truth for credential locations. This page records the categories only; never the values.

  • Render env vars — every prod secret used by the Flask app + crons.
  • 1Password — canonical store for human-typeable credentials (logins, master passwords).
  • Internal-host on-disk secrets — local files at well-known paths for service runtime (mirrored to Render env where needed).

Where the data lives

All app-authored data lives in SQLite at /var/data/cenas_kitchen.db on the Render persistent disk. SQLAlchemy is the ORM. Tables are grouped logically below; the schema is in app/models.py.

External-facing endpoints

  • app.cenaskitchen.com/healthz — liveness probe.
  • app.cenaskitchen.com/ezcater/webhook — EZCater inbound order webhook (signature-verified). Auto-creates a DriverAssignmentJob row + POSTs a wake to the AiCk gateway.
  • app.cenaskitchen.com/<store>/catering/assign_driver + /status — manager dropdown re-assign + polling endpoint (per-store-scoped).
  • app.cenaskitchen.com/catering/assign_driver/result — gateway callback (flat route, EXEMPT_PREFIXES'd, X-Cena-Token gated).
  • app.cenaskitchen.com/partner/* — partner dashboard.
  • app.cenaskitchen.com/<store>/* — store manager surfaces.
  • app.cenaskitchen.com/sam/* — operator-only surfaces.

Internal-facing endpoints (Tailscale)

  • http://100.108.119.19:8765/jobs/driver-assign — AiCk gateway, X-Cena-Token, accepts {job_id, order_id, new_driver, current_driver?, callback_url}; returns 202 + async runs Selenium UI flow.
  • http://100.108.119.19:8765/cena/stream + /cena/wake-on-chat — Cena runtime entry points from /sam/chat.
  • http://100.73.38.82:8766/jobs/driver-assign — Mini_IT13 (ck) failover; identical contract to AiCk gateway.
  • http://192.168.1.134:8765/partner/developer/chat/messages.json — LAN team chat hub (Mini_IT13).

Deployment surface

The single deployable artifact is the Flask app on Render. Every push to origin/main triggers a Render auto-deploy (~2–3 minute swap). The crons share the same deploy image, so a single git push updates the web service and all background jobs together.
