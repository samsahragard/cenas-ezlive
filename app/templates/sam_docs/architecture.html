Architecture

High-level system design + the operational map of every account, machine, and data source.

System at a glance

The platform is a Flask application served from a Render web service, sitting behind a Cloudflare-fronted partner gate, with a SQLite database on a persistent disk, scheduled cron services for background work, and several outbound integrations.

  • Web app — app.cenaskitchen.com, Render-hosted Flask + Jinja templates.
  • Database — SQLite on Render's persistent disk at /var/data/cenas_kitchen.db; SQLAlchemy ORM + boot-time create_all for migrations (Alembic not wired on Render).
  • Background jobs — Render cron jobs (Toast refreshes, anomaly briefs, calendar rolls, event-traffic refresh, sales insights, etc.).
  • External integrations — Toast (POS + analytics; also the NEW scheduling source via /labor/v1/shifts), Sling (scheduling — BEING REPLACED by Toast scheduling per Sam #1010, cutover in progress), EZCater (catering intake; webhook + IMAP + Playwright UI flow), vendor email channels, Sentry (error monitoring).
  • AI surface — Cena gateway (HTTP + Tailscale) on AiCk; /sam/chat talks to it for the Sam-facing assistant. Anthropic Claude (Opus/Sonnet) is the tool-enabled provider via the Cena gateway; Google Gemini (Flash) is plain-chat only and routes directly to Google, bypassing the gateway and its tool schema (sam_chat.py). Active model is Sam-selected (cena_active_model.txt).
  • Mobile shell — Capacitor WebView app com.cenaskitchen.app ("Cenas Kitchen Employee", Play Console internal testing) that loads app.cenaskitchen.com directly; built + signed in GitHub Actions (upload keystore via GitHub Secrets). The signed AAB is produced as a CI artifact and uploaded to Play Console manually — there is no Android Publisher API step. A debug APK is auto-published to a GitHub release for driver sideloading (/driver/app.apk).
  • Driver-assignment UI flow — webhook auto-assigns + manager dropdown re-assigns drive the partnerportal.ezcater.com UI on internal hosts. As of 2026-05-27 (Sam #1153) the PRIMARY is pwck (Mini_IT13:9000, Playwright); aick's Selenium gateway is fallback. Render only creates job rows + POSTs wakes; never touches Chrome itself. (Render env: CENA_GATEWAY_URL=pwck primary, PWCK_PRIMARY_URL split out so it no longer collides with the Cena-chat gateway.)

Operational inventory

  • Cloudflare — DNS + Tunnel for internal-service exposure.
  • Tailscale — VPN mesh across machines.
  • GitHub — source control; SSH deploy key authorizes pushes.
  • Anthropic — API key; primary AI provider.
  • Google — Gemini API + Workspace.
  • Toast — POS account + Analytics OAuth app.
  • Sling — scheduling (BEING REPLACED by Toast scheduling API; key slated for removal post-cutover).
  • EZCater — catering intake.
  • Sentry — errors.
  • 1Password — RETIRED 2026-05-28 (Sam directive). Credential master index is the samai desktop file (credential_inventory_2026-05-27.md); operational secrets live in two on-disk lanes on AiCk (app/runtime + Cena gateway) with the legacy openclaw store archived. See the Pass tab for exact locations.

Machines

  • Render web service — production app + cron runtime. SQLite lives on the Render persistent disk; restarting the service does NOT wipe data.
  • AiCk (Mini_IT12, 100.108.119.19) — internal host. Runs the Cena gateway (port 8765, exposed via Cloudflare Tunnel + reachable on the tailnet), the aick Selenium driver-assign worker (now FALLBACK behind pwck), the watchdog (port 8767, docck-callable), the CenaHeartbeatSender → docck, and the persistent /sam/chat ↔ Cena local chat (port 8770, Tailscale-only). All run as Windows Scheduled Tasks with auto-restart (CenaGateway, CenaChatWatcher, CenaHubRelay, CenaSamChat, CenaTaskPinger, CenaHeartbeatSender, ck-watchdog/CenaWatchdog).
  • Mini_IT13 (ck's box, 100.73.38.82) — peer internal host. Runs the LAN team chat hub (port 8765, hub.py) + the online-dev-chat↔hub bridge (bridge.py under run_bridge.py; the separate Telegram↔hub bridge runs on AiCk, see Integrations), pwck the PRIMARY driver-assign gateway (port 9000, Playwright), ck_gateway the driver-assign FAILOVER (port 8766 — UP, netstat-verified 2026-05-28), the ck watchdog (port 8767), and the PwckHeartbeatSender → docck. (A separate "cena2 active/active Layer-4 peer" concept is unbuilt — not to be confused with the :8766 failover, which IS running.)
  • Operator phone — Tailscale member; reads Telegram via @Cenasbot for dev-chat traffic (bridge mirrors hub ↔ Telegram).

Deployment surface

The single deployable artifact is the Flask app on Render. Every push to origin/main triggers a Render auto-deploy (~2–3 minute swap). The crons share the same deploy image, so a single git push updates the web service and all background jobs together.
