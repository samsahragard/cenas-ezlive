Architecture

High-level system design + the operational map of every account, machine, and data source.

System at a glance

The platform is a Flask application served from a Render web service, sitting behind a Cloudflare-fronted partner gate, with a Postgres database, scheduled cron services for background work, and several outbound integrations.

  • Web app — app.cenaskitchen.com, Render-hosted Flask + Jinja templates.
  • Database — Postgres on Render (managed); SQLAlchemy ORM + Alembic-style migrations baked into create_all.
  • Background jobs — Render cron jobs (Toast refreshes, anomaly briefs, calendar rolls, event-traffic refresh, sales insights, etc.).
  • External integrations — Toast (POS + analytics), Sling (scheduling), EZCater (catering intake), vendor email channels, Sentry (error monitoring).
  • AI surface — model gateway for operator chat (/sam/chat) and the team-internal assistant.

Operational inventory

Accounts (top-level)

  • Render — hosts the web service + crons + Postgres.
  • Cloudflare — DNS + Tunnel for internal-service exposure.
  • Tailscale — VPN mesh across machines.
  • GitHub — source control; SSH deploy key authorizes pushes.
  • Anthropic — API key; primary AI provider.
  • Google — Gemini API + Workspace.
  • Toast — POS account + Analytics OAuth app.
  • Sling — scheduling.
  • EZCater — catering intake.
  • Sentry — errors.
  • 1Password — canonical credential vault (CLI integration available).

Machines

  • Render web service — production app + cron runtime.
  • Primary internal host — runs the operator chat backend, gateway services, and Cloudflare Tunnel client. Tailscale-bound for internal access.
  • Peer internal host — secondary gateway, message-queue worker, cross-host sync. Tailscale-bound.
  • Operator phone — Tailscale member for on-the-go access to internal services.

Credential locations

The Pass tab is the source of truth for credential locations. This page records the categories only; never the values.

  • Render env vars — every prod secret used by the Flask app + crons.
  • 1Password — canonical store for human-typeable credentials (logins, master passwords).
  • Internal-host on-disk secrets — local files at well-known paths for service runtime (mirrored to Render env where needed).

Deployment surface

The single deployable artifact is the Flask app on Render. Every push to origin/main triggers a Render auto-deploy (~2–3 minute swap). The crons share the same deploy image, so a single git push updates the web service and all background jobs together.
