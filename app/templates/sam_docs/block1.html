Block 1 — Foundation Shipped

The cross-cutting primitives every Block 2 and Block 3 feature depends on. Block 1 is closed; amendments only via formal spec amendment citing commit SHA.

Block 1 in one sentence

Block 1 establishes the data primitives, the cross-page UI primitives, and the cross-cutting signal primitives — so subsequent blocks build features without re-inventing infrastructure each time.

Preconditions

Role Taxonomy

The canonical role enumeration for the system. Every authenticated session resolves to exactly one role; the role-to-permission map is the source of audience eligibility. The 15 live roles (permissions.py ROLE_PERMISSIONS): partner, corporate, gm, km, assistant_km, corporate_chef, prep_manager, foh_manager, expo, driver, plus 5 hourly (cook / server / busser / host / bartender). (operator is an env-var gate, NOT a role-map entry.) The Permissions tab covers the full model.

ScheduledEvent

The unified scheduled-event primitive — a single table model that backs reservations, deliveries, catering events, manager rituals, and any other time-bounded event in the system. Replaces the prior fragmented per-feature tables. Carries: start/end timestamps, store, type, status, owner, payload (typed via JSON).

1A — Task System

Task model + escalation policy + audit log. Tasks are the primary work-tracking primitive: they have a status (open / in_progress / completed / cancelled), an owner, a store scope, an optional deadline, and an audit trail of every state transition. The escalation cron runs hourly and bumps overdue tasks per the per-type escalation rules.

1B — Ribbon Component

The shared in-page ribbon control — a unified header / breadcrumb / status bar component used across the partner dashboards. Single Jinja partial + a scoped style block. All dashboards adopt it for a consistent ribbon UX.

1C — Ribbon Router

Deep-link routing through the ribbon: any ribbon item can be addressed via URL parameters and the router resolves the page state accordingly. Enables shareable links into specific dashboard tabs / filters / time ranges.

1F — Sales Insights

The Toast-derived sales rollup surface. Cron-refreshed (~30 min cadence) sales windows that feed the dashboards. Covers per-store, per-period (today / week / month / YTD), revenue, ticket count, average ticket, and labor cost percentage.

1G — Team Tab

Per-store team management — roster, shift status, contact info, role/store assignment. The schedule view is now Toast-shift-derived (Sling-compatible labels; Sling is being replaced — see Phase 2 Roadmap); manual overrides land in the per-store team table.

1H — Pay Masking

Privacy controls on driver pay surfaces. Pay amounts on shared surfaces (live tracking, pay history visible to non-owners) display masked; full amounts surface only to the row owner and to permitted roles.

1J — AmbientSignal Refactor

Cross-cutting signal primitives. The ambient-signal system standardizes how lightweight notifications, status badges, and background-cron-derived state surface across the UI. Single signal table + a Jinja partial that any page can include.

Block 1 cross-references

  • Tasks (1A) consume ScheduledEvent (precond) for deadline scheduling.
  • The ribbon (1B) + router (1C) front every dashboard surface from Block 2 onward.
  • Sales Insights (1F) feeds Today / Operations / Catering dashboards.
  • Team Tab (1G) consumes per-store roster + ScheduledEvent for shift state.
  • Pay Masking (1H) gates pay display per the role/audience map (see Permissions).
  • AmbientSignal (1J) underlies Notifications, anomaly alerts, and cron-status badges.
