Cena Surface Active

The operator-only AI assistant. The surface, the tools, and the escalation policy.

What it is

Cena is the operator-facing AI assistant: a runtime on an internal host (cena_gateway.py on AiCk) that exposes a chat surface on the web app at /sam/chat, and operates the system on the operator's behalf with a defined tool catalogue and audit log. Model: the gateway defaults to claude-sonnet-4-6 (cena_gateway.py:113) and also exposes a picker for claude-opus-4-7 + gemini-2.5-flash / gemini-3.5-flash.

Audience gate

The Cena chat surface is gated to a single user: the operator whose user id matches the SAM_CHAT_USER_ID env var. The capability is NOT role-inherited — even a partner / admin role does not unlock it without the explicit user-id match. When the env var is unset, the surface is dormant and inaccessible to everyone.

Tool catalogue

Cena operates the system via a defined tool surface. Tools are written as JSON-schema-typed endpoints on the gateway; each call is logged to the audit table. Tool categories:

  • Read tools — DB probes, account lookups, vendor / catering / order reads.
  • Action tools — vendor-order operations, task creates, status updates.
  • Integration tools — Render env get/set, Render deploy, Telegram-send, dev-chat post.
  • Coordination tools — message-queue ops, hub-read, gateway peer sync.
  • Restart tool — agent-restart, single-use, permission-gated per call (no standing grant).

SQL safety

Read-only DB probes pass through a three-layer parse gate before execution:

  1. Parse + count — the input must parse as a single statement.
  2. Shape + write-ban — the statement must be a SELECT (DML rejected: INSERT / UPDATE / DELETE / DDL all blocked).
  3. Column denylist + exempts — columns containing secrets (hashed passwords, tokens) are denied unless the call is in the exempt whitelist for legitimate ops.

Audit log

Every tool call writes a row to the audit table: which tool, what arguments, what return, who fired it, when. The audit is read-only via a separate operator-only endpoint; tools cannot rewrite it.

Escalate-on-uncertainty

When the assistant is uncertain about an action that would be destructive or hard to reverse, it stops and asks the operator. The pattern:

  • Any irreversible action: explicit operator confirmation required, per-call (no standing "yes, do this kind of thing").
  • Restart of another assistant: requires a fresh operator approval in the operator chat with the exact phrase the tool expects (single-use; no carry-over).
  • Destructive DB action: explicit confirmation + dry-run summary before execution.

Persistent chat (local)

The operator ↔ Cena local chat at port 8770 on the primary internal host persists conversation state independently of the web chat. This is the channel Cena uses to continue work through operator disconnects: when the operator drops off the web, Cena can pick up the conversation from the local chat next time they reconnect.

External reachability

Internal-only by design. The gateway binds to Tailscale only; external traffic enters via a Cloudflare Tunnel terminator. There is no public-internet binding on the gateway port. The web app at /sam/chat is the public-facing entry point and is gated on user id.
